YOLOv2

Preface

这是一篇承上启下的论文，相对而言非常重要。

B站讲解：https://www.bilibili.com/video/BV1Q64y1s74K

论文地址：https://arxiv.org/abs/1612.08242

现在开始看YOLOv2了，感觉v1毕竟读了那么长时间了，也了解的差不多了。凎！我们先了解一下算法的具体算法思路吧，然后开始读论文学习相关的内容。

YOLO9000引入了一系列改进，加入了anchor,加入了batch normalization， 加了多尺度的训练这些，这些对于之后的目标检测有着非常重要的影响，需要我们好好思考学习一下。

我们可以看一下YOLO大致发表的版本：

再来张我们各个目标检测比较有影响力模型：

  • YOLOv1的虽然速度很快，因为它是单阶段算法嘛，但是他的识别mAP相对于我们其他的先进的两阶段来看，还是有挺大的差距。
  • 另外就是定位性能比较差，相对于FasterRCNN来看，它精确调整框框的位置的能力不是很好。
  • 同时来看，它的recall比较低，全部目标全部检出的能力比较差。
  • YOLOv2对以上做了很大的改进
  • YOLO9000名字叫做：YOLO9000：Better, Faster, Stronger, 这里的Better就是更加的准确.

这里有着许多的手段来增加我们的准确率，如上总结来看，非常厉害了。我们一一总结来看。

Better

  • *”Error analysis of YOLO compared to Fast R-CNN shows that YOLO makes a significant number of localization errors. Furthermore, YOLO has relatively low recall compared to region proposal-based methods. Thus we focus mainly on improving recall and localization while maintaining classification accuracy.”*

相比于Faster RCNN,YOLOv还是在定位上，缺陷很大，毕竟我们YOLOv1是直接预测的框的位置，而Faster-RCNN是通过预测的offset（因为提前有proposal了，所以说只需矫正一下框就OK了。)

CV的大势是更大更深的网络，更好的表现经常是由于更深的网络训练以及更多数据来支撑。这里YOLO为了保证速度，反而简化了设计的神经网络。

High Resolution Classifier

一般神经网络都是以很小的图片进行检测与训练的，但是YOLOv1的输入图像是448×448，它需要从小分辨率切换到大分辨率，这么一来无疑就损失了许多，得到了许多没用的东西，所以我们选择使用直接让YOLOv2在448×448的高分辨率的图像进行训练。

可以看到这么一来我们能够增加3.5%mAP的精确率。网络不需要从小分辨率到大分辨率去适应了，我们直接进行了相关的大分辨率的预训练。

Convolutional With Anchor Boxes

这里使用了Anchor，后来直到v5都在使用anchor,虽然之后不用了，变成了anchor-free, 但是理解这些还是非常重要的。我们就从这里开始了解一下Anchor。

就像前面说的，YOLO直接预测框的x,y,w,h，但是FasterRCNN的RPN预测的是与anchor的offset！！（呜呜呜我真是个废物以为自己掌握很清楚的Faster RCNN却被YOLO作者无意中对比打了脸，看来自己掌握的还是太浅了，源码读了个寂寞啊，赶快抢救下）

预测offsets而不是直接预测坐标coordinates可以简化问题从而让我们的网络更好的去学习，这个很好理解，因为有了anchor相当于有了一个参照物，神经网络只需要继续让这个框变得 更加精确就OK了，但是如果从头开始预测需要很长时间的变化与初始化运行。

作者去掉了YOLOv1中的全连接层，然后使用anchor来预测我们的结果。此外也去掉了一个pooling层来使网络输出的分辨率更高。另外作者将输入的shape变成了416×416， 原来是448. 这是因为作者想要得到一个奇数从而让图片的中心是一个cell而不是4个，因为很多我们需要检测出的object都会出现在图片中心，所以这么一来会更加方便与合理，只需要检测一个大框而不是有4个cell来检测输出结果。

YOLO将图片抽缩32倍所以我们最后得到的是13×13的feature map。

*”When we move to anchor boxes we also decouple the
class prediction mechanism from the spatial location and
instead predict class and objectness for every anchor box.
Following YOLO, the objectness prediction still predicts
the IOU of the ground truth and the proposed box and the
class predictions predict the conditional probability of that
class given that there is an object.”*

v2好像训练阶段也是每一个box都预测了相应的类别概率。

使用anchor boxes准确率有一个小小的下降，但是recall却上升了。之前是69.5mAP, 81%recall，现在变为69.2mAP, 88%recall.

当然之前类别是归grid cell管的，现在变为归anchor管了。之前YOLOv1中是每一个grid cell的B个box中只能有一个类别，并且只能有一个类别输出矩阵（根据IOU），但是现在变成归生成的每个anchor管之后，一个grid cell会有多个anchor并且很有可能可以预测出不同的类别，也就是没有了一个gird cell一个物体的限制。

等详细理解了自己再写一遍了解一下。

Dimension Clusters

*”We encounter two issues with anchor boxes when using them with YOLO.”*

使用anchor的时候，作者遇到了两个问题。

hand picked priors

*”The first is that the box dimensions are hand picked. The network can learn to adjust the boxes appropriately but if we pick better priors for the network to start with we can make it easier for the network to learn to predict good detections.”*

我们的anchor box是人为标注的，虽然这么做网络仍然可以通过预测offset的方式来优化修正，但是如果我们能够最初就给出一个比较合适的anchor，那么网络无疑可以给出更好更快的训练结果。

所以作者对于数据集的GT做了K-means聚类，当然作者并没有做标准的聚类，因为最标准的聚类采用的是欧几里得距离，显然这里如果采用欧几里得距离会有很大的问题，因为对于大的边框和小的边框来说相同的欧几里得误差是不公平的，作者这里定义了一种全新的距离：

d(box, centroid) = 1 − IOU(box, centroid)

这就和框的尺寸没有关系了，这是纯粹的IOU的占比，当然这个距离越小，我们的IOU占比越大，这就合理很多。

作者尝试了很多个k值，最后选用k=5作为一个比较好的能够兼顾模型复杂度与高recal的tradeoff。

作者也给了一个表格用来对比不同聚类k，不同聚类方法，是否聚类的效果：

最后得出了使用聚类确实提高了效果这样的一个思路。

Direct location prediction

当用anchor box这种方法的时候，作者遇到的第二个问题在于模型的发散，模型不是很稳定。其中着重提到了在训练迭代的早期。大多数的instability是由于预测box的(x, y)的时候，在RPN中网络预测的是$t_x, t_y$​, (RPN中)我们的(x, y)中心坐标计算的方法是：

x = (tx ∗ wa) − xa
y = (ty ∗ ha) − ya

这其中的$t_x, t_y$就是我们的预测偏移量，然后$w_a, h_a$是anchor的宽高，如果这里$t_x=1$，那么相当于box向右移动一个box的宽的距离。-1向左移动。这么做的话就对于最终预测出来的anchor box没有任何限制，它可以出现在我们的图片的每一个位置，这显然是有些不合理的，因为让这个grid cell负责这些预测就是因为中心点在这里面，这反而很容易跑到外边就对模型的预测初始迭代这些都很不好。随机初始化状态下模型需要花费很长时间去稳定，才能预测出一个比较合理比较sensible的模型。

所以作者最后选择将grid cell的大小归一化到0与1之间然后选择使用一个logistic activation来限定我们的预测在grid cell的范围内。

*”The network predicts 5 bounding boxes at each cell in
the output feature map. The network predicts 5 coordinates
for each bounding box, tx, ty, tw, th, and to. If the cell is
offset from the top left corner of the image by (cx, cy) and
the bounding box prior has width and height pw, ph, then
the predictions correspond to:”*

$b_x = σ(t_x) + c_x$

$b_y = σ(t_y) + c_y$

$b_w = p_we^{t_w}$​

$b_h = p_he^{t_h}$

$Pr(object) ∗ IOU(b, object) = σ(to)$

我们可以看到我们每个bbox都会predicts5个参数：$t_x, t_y, t_w, t_h, t_o$.我们可以先来看一下以上这些个参数的意思：

$c_x,c_y$ 指的是该cell的左上角离我们的image的左上角的x,y轴方向的偏移量，也就是左上角的坐标。然后$p_w,p_h$​就是我们的anchor的基本宽和高，然后$b_x, b_y,b_w, b_h$是我们的预测框的x,y坐标以及高宽。

对这方面做一个总结来看，由于我们做了一个限制使我们的预测location更加确定从而使得我们的模型快速稳定，同时使用聚类来让我们的模型训练更加轻松准确，最终使我们的YOLO提升了几乎5%个点。

Fine-Grained Features

*”Our detector runs on top of this expanded feature map so that it has access to fine grained features.”*

Fine-Grained Features, 也即“细粒度特征”。

实际上也就是将13×13×1024的前一步卷积层得到的26×26×512的变成13×13×2048然后和原来的13×13×1024的original features合并到一块儿进行训练而已，作者也说了类似于ResNet的思路，这个大概有1%的提升，实际上提升并不是非常大。不像真正的resnet一样是一种非常大的提升。

但是代码上的处理和论文上的处理不太一样，代码上的处理是将26×26×512通过1×1Conv变为26×26×64然后再拆分成13×13×256再和之前的合并.

Multi-Scale Training

*”The original YOLO uses an input resolution of 448 × 448. With the addition of anchor boxes we changed the resolution to 416×416. However, since our model only uses convolutional and pooling layers it can be resized on the fly. We want YOLOv2 to be robust to running on images of different sizes so we train this into the model.”*

作者不在固定输入图像尺寸的大小了，由于我们的网络层是降采样32倍，所以我们选择每10个batch就随机选取一种图片尺寸，这个尺寸就是32的倍数：{320， 352，384，……，608}，相当于让我们的模型更加robust。会强迫神经网络学习去预测不同输入size的图片。

接着就是作者的”吹嘘“（但这是有资本有实力的吹）

*”At low resolutions YOLOv2 operates as a cheap, fairly accurate detector. At 288 × 288 it runs at more than 90 FPS with mAP almost as good as Fast R-CNN. This makes it ideal for smaller GPUs, high framerate video, or multiple video streams. At high resolution YOLOv2 is a state-of-the-art detector with 78.6 mAP on VOC 2007 while still operating above real-time speeds. See Table 3 for a comparison of YOLOv2 with other frameworks on VOC 2007.”*

或许你会问为什么我们可以输入不同大小的图片而不改变我们的模型结构？这是因为我们这里并没有选择使用我们的全连接层，这里YOLOv2将模型末尾的两个全连接层去掉了，然后选择使用了一个Avgpool，YOLOv2用的实际上是自己研究出来的darknet(会在Faster那一节讲)，这就导致我们的不管输入是什么，输出的结果都只取决于最后的通道数。

运算符概述

运算符分类

  • 纯单目运算符，只能有一个操作数，包括：!、~、sizeof、new、delete 等
  • 纯双目运算符，只能有两个操作数，包括：[ ]、->、%、= 等
  • 三目运算符，有三个操作数，如“? :”
    既是单目又是双目的运算符，包括：+、-、&、* 等
  • 多目运算符，如函数参数表 “( )”。
  • 左值运算符是运算结果为左值的运算符，其表达式可出现在等号左边，如前置 ++、– 以及赋值运算 =、+=、*= 和 &= 等。右值运算符是运算结果为右值的运算符，如 +、-、>>、 %、后置 ++、– 等。
  • 某些运算符要求第一个操作数为左值，如 ++、– 、=、+=、&=等。

我们先看个例子看一下传统左值运算符的用法：

#include 
int main(int argc, char *argv[])
{
    int x = 0;
    ++x;                     //++x 的结果为左值（可出现在等号左边）
    ++++x;                   //++x 仍为左值，故可连续运算，x=3
    --x = 10;                //--x 仍为左值，故可再次赋值，x=10
    (x = 5) = 12;            //x=5 仍为左值，故可再次赋值，x=12
    (x += 5) = 7;            //x+=5 仍为左值，故可再次赋值，x=7
    printf("%d %d", x, x++); //( )可看作任意目运算符
    return 0;
} //(x--)++是错的：x--的结果为右值，而++要求一个左值

左值右值

这里可能很多人都忘了（比如我自己）什么是左值，什么是右值。这里可以参考博文，
我们这里可以简单展开讲一下：
右值代表一个临时的值，可以理解为临时变量也就是常量，比如说1, 2, “slkjdf”像这种不可以被赋值处理的，而左值是占有特定内存空间并且可以被赋值的一个变量。
右值只存在于每一个计算过程中的临时寄存器中。

我们简单看几个比较重要的例子：
首先我们如果在编译器中编译这么几行代码：

int foo() {return 2;}

int main()
{
    foo() = 2;

    return 0;
}

这里编译器会报错：

test.c: In function 'main':
test.c:8:5: error: lvalue required as left operand of assignment

这里会说，左值运算符需要一个lvalue（左值），而我们函数返回了右值无法接受=2这样一个赋值。
右值不能接受赋值操作，赋值语句的左边应当是一个左值（这或许就是左值名称的又来）

另外如果我们编译返回引用的这样一个函数：

int& foo()
{
    return 2;
}

编译器仍然会报错：

testcpp.cpp: In function 'int& foo()':
testcpp.cpp:5:12: error: invalid initialization of non-const reference of type 'int&' from an rvalue of type 'int'

因为我们这里定义了有关foo()函数返回一个引用，但是返回的确实一个右值2，2这样一个临时变量很明显不可以做为一个引用使用（它在内存中没有固定的地址），这自然是不对的。

另外这也给我们一点启发，函数的返回值是完全可以赋值的，只要让它的返回类型为引用类型的就可以(奇怪的C++知识)：

int globalvar = 20;

int& foo()
{
    return globalvar;
}

int main()
{
    foo() = 10;
    return 0;
}

运算符重载

C++预定义了简单类型的运算符重载，如3+5、3.2+5.3分别表示整数和浮点加法。故C++规定运算符重载必须针对类的对象，即重载时至少有一个参数代表对象 (类型如A、const A、A &、const A &、volatile A等) 。

这里再次强调，类的重载必须针对类的对象来进行，而对于我们普通的整数的加减这种操作，也同样是重载，是C++预定义好的不用我们再去定义的重载类型。

C++用operator加运算符进行运算符重载。
对于普通运算符成员函数，this 隐含参数代表第一个操作数对象。

运算符重载分类

运算符分为不能重载的、只能重载为普通函数的、只能重载为类普通函数的、都可以被重载的。但是这一切都不可以被重载为静态类函数成员。

根据能否重载及重载函数的类型，运算符分为：

  • 不能重载：sizeof、. 、.*、::、? :
  • 只能重载为类的普通成员函数：=、–>、( )、[ ]
  • 只能重载为普通函数：new、delete
  • 其他运算符：可以重载为类的普通成员函数和普通函数，但不能重载为类的静态成员函数。

可以参考一下微软参考文档关于运算符重载一般规则

一些重载时的注意点

若运算符为左值运算符，则重载后运算符函数最好返回非只读引用类型(左值)。当运算符要求第一个参数为左值时，不能使用 const 说明第一个参数 (含this)，例如 ++、–、=、+= 等的第一个参数。 也就是我们不能对于需要左边参数为左值的运算符重载后左边参数为const类型，因为这样会产生冲突。类普通函数的重载一般第一个参数都是this指针，所以重载函数后面有时不可以加const

重载运算符函数可以声明为类的友元；重载的普通运算符成员函数也可定义为虚函数；重载的非成员函数被视为普通函数。

重载运算符函数一般不能缺省参数，只有任意目的运算符 ( ) 省略参数才有意义。

重载不改变运算符的优先级和结合性。

重载一般也不改变运算符的操作数个数。特殊的运算符 ->、++、–除外。

可以看一些例子：

#include
using namespace std;

class A;
int operator = (int, A &);           //错误, 不能重载为普通函数
A &operator +=(A &, A &);      //正确，重载 += 为普通函数
A &operator ==(A *a, A b[ ]);   //A * 和 A[ ] 参数不代表对象
class A {
        friend int operator=(int, A &);            //错误，不存在普通函数operator=( )
        static int operator( )(A &, int);            //错误，不能为静态成员
        static int operator+(A &, int);             //错误，不能为静态成员
        friend A & operator += (A &, A &);   //正确， operator += ( ) 是什么函数？
        A &operator ++( );    //隐含参数this代表一个对象
};

这里可以看到=操作符是不可以被重载为普通函数的，重载为类内普通函数就问题。注意这里+=虽然同样要求第一个操作数为左值，但是就可以重载为普通函数！
另外注意所有重载运算符都不可定义为静态函数。

Preface

“This tutorial is intended to be a gentle introduction to argparse, the recommended command-line parsing module in the Python standard library.“

官方文档：https://docs.python.org/3/howto/argparse.html

新生招新群里有人问了一些问题，他给出的问题代码中包括这个库，目前我目测是因为他没有用好这个库导致图片读取返回了None，正好我也早就想要去了解一下这个库，顺便学一下。

了解完后发现我的推测是正确的。。。

Concepts

作者首先是介绍了一下我们的一些有关概念，他使用了Linux常用命令ls来引入：

$ ls
cpython  devguide  prog.py  pypy  rm-unused-function.patch
$ ls pypy
ctypes_configure  demo  dotviewer  include  lib_pypy  lib-python ...
$ ls -l
total 20
drwxr-xr-x 19 wena wena 4096 Feb 18 18:51 cpython
drwxr-xr-x  4 wena wena 4096 Feb  8 12:04 devguide
-rwxr-xr-x  1 wena wena  535 Feb 19 00:05 prog.py
drwxr-xr-x 14 wena wena 4096 Feb  7 00:59 pypy
-rw-r--r--  1 wena wena  741 Feb 18 01:01 rm-unused-function.patch
$ ls --help
Usage: ls [OPTION]... [FILE]...
List information about the FILEs (the current directory by default).
Sort entries alphabetically if none of -cftuvSUX nor --sort is specified.
...
  • ls 在没有任何options的时候默认展示目前目录下所有内容
  • 如果我们想要他不仅仅提供它默认的选项的时候，我们需要加一点其他的内容，也就是其他的参数，例如加一个 ls pypy
  • 我们如果多加一个参数-l那便是展示出了文件的一些更为详尽的信息
  • *”That’s a snippet of the help text. It’s very useful in that you can come across a program you have never used before, and can figure out how it works simply by reading its help text.”*

我们可以开始介绍Argparse库了。

The basics

我们以一个非常简单的例子开始（这个例子几乎什么都没做）

import argparse
parser = argparse.ArgumentParser()
parser.parse_args()

然后我们在Linux shell里输入命令跑一下这个程序：

$ python3 prog.py
$ python3 prog.py --help
usage: prog.py [-h]

optional arguments:
  -h, --help  show this help message and exit
$ python3 prog.py --verbose
usage: prog.py [-h]
prog.py: error: unrecognized arguments: --verbose
$ python3 prog.py foo
usage: prog.py [-h]
prog.py: error: unrecognized arguments: foo

我们来看看发生了什么：

  • 运行这个脚本时如果不带任何参数那么就会没有任何输出到stdout， 不是非常有用
  • 第二个开始展示我们argparse的用处，我们几乎什么都没做，但是却得到了许多有用的帮助文档。
  • --help参数，可以缩短为-h。

Introducing Positional arguments

再来一个例子：

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("echo")
args = parser.parse_args()
print(args.echo)

同样的在Linux上运行：

$ python3 prog.py
usage: prog.py [-h] echo
prog.py: error: the following arguments are required: echo
$ python3 prog.py --help
usage: prog.py [-h] echo

positional arguments:
  echo

optional arguments:
  -h, --help  show this help message and exit
$ python3 prog.py foo
foo
  • 我们加了一个新的方法add_argument(), 这个就是我们是用来指定程序将会接受哪一个命令行指令。在我们这种情况下，我们命名它为echo, 以便于他是命令行的函数。
  • 现在启动我们的程序需要特别指定一个参数
  • parse_args()方法返回的来自我们指定参数的一些数据，当前情况下，是echo
  • 这个变量也是某种“魔术”方法， 不需要指定哪个数据被存到哪个地方。我们也能注意到它的名字匹配对应了echo方法。

但是我们发现，虽然这样看起来还不错，但是事实上我们还可以进一步改进，我们了解一个参数的功能还是只能从名字中猜测或者是从源代码中看。显然这是很难被我们接受的，所以我们可以考虑使用以下命令：

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("echo", help="echo the string you use here")
args = parser.parse_args()
print(args.echo)

在Linux shell中运行：

$ python3 prog.py -h
usage: prog.py [-h] echo

positional arguments:
  echo        echo the string you use here

optional arguments:
  -h, --help  show this help message and exit

现在我们甚至可以让它更加useful:

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("square", help="display a square of a given number")
args = parser.parse_args()
print(args.square**2)

Following is a result of running the code:

$ python3 prog.py 4
Traceback (most recent call last):
  File "prog.py", line 5, in 
    print(args.square**2)
TypeError: unsupported operand type(s) for ** or pow(): 'str' and 'int'

可以发现这似乎不是很奏效，程序报错了，很好理解：因为这里面从命令行读取的是字符串，显然字符串是没有办法做平方的。

但是，作者后来又加了一句：unless we tell it otherwise. So, let’s tell argparse to treat that input as an integer:

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("square", help="display a square of a given number",
                    type=int)
args = parser.parse_args()
print(args.square**2)

接着，我们就可以看到了：

$ python3 prog.py 4
16
$ python3 prog.py four
usage: prog.py [-h] square
prog.py: error: argument square: invalid int value: 'four'

Introducing Optional arguments

到目前为止我们所使用的还都是一些positional arguments，现在我们加一些optional ones（原文中指代arguments）。

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("--verbosity", help="increase output verbosity")
args = parser.parse_args()
if args.verbosity:
    print("verbosity turned on")

输出为：

$ python3 prog.py --verbosity 1
verbosity turned on
$ python3 prog.py
$ python3 prog.py --help
usage: prog.py [-h] [--verbosity VERBOSITY]

optional arguments:
  -h, --help            show this help message and exit
  --verbosity VERBOSITY
                        increase output verbosity
$ python3 prog.py --verbosity
usage: prog.py [-h] [--verbosity VERBOSITY]
prog.py: error: argument --verbosity: expected one argument
  • 这个程序会在我们指定 --verbosity这个参数的时候才会展示出一些东西，在没有这个参数的时候什么都不会展示。
  • 为了说明这个参数命令是可选的，我们如果没有加这个参数程序仍然可以运行而不报错，注意默认状态下，该参数是不被取用的。
  • 注意这个--help的信息关于 --verbosity是不一样的。
  • 当我们使用--verbosity选项的时候， 我们必须指定一些特定的值赋给该项参数。

以上参数--verbosity可以接受任意整数值, 但是通过看代码我们可以发现，实际上只有两个值是真正有用的，True or False. 我们可以相应的修正一下我们的代码：

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("--verbose", help="increase output verbosity",
                    action="store_true")
args = parser.parse_args()
if args.verbose:
    print("verbosity turned on")

看一下运行情况：

$ python3 prog.py --verbose
verbosity turned on
$ python3 prog.py --verbose 1
usage: prog.py [-h] [--verbose]
prog.py: error: unrecognized arguments: 1
$ python3 prog.py --help
usage: prog.py [-h] [--verbose]

optional arguments:
  -h, --help  show this help message and exit
  --verbose   increase output verbosity
  • 可以看到，现在该参数更多的像是一个flag, 而不是一个需要接受的值values。我们甚至改变选项的名字来匹配我们的idea. 注意我们现在使用了，或者说指定了一个新的关键字：action. 并且给它了一个默认的值store_true. 这就意味着，我们选择了这个参数，类似于这个命令python3 prog.py --verbose. 而不是指定他代表False.
  • 当我们给它指定一个值的时候，他会报错
  • 注意这里的help text就不一样了。可以对比一下上面的那个python3 prog.py --help和我们这里的python3 prog.py --help. 就可以看出区别。

Short options

如果对命令行的一些命令比较熟悉，我们就可能会想一下是不是这些参数都应该有一个快捷键呢？这是确实得，而且是非常简单的，只需要前面加一个类似于-v的指令就行：

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("-v", "--verbose", help="increase output verbosity",
                    action="store_true")
args = parser.parse_args()
if args.verbose:
    print("verbosity turned on")

output:

$ python3 prog.py -v
verbosity turned on
$ python3 prog.py --help
usage: prog.py [-h] [-v]

optional arguments:
  -h, --help     show this help message and exit
  -v, --verbose  increase output verbosity

注意到我们的help text里面也加了-v这个命令。

Combining Positional and Optional arguments

我们可以看一下如何将这两种参数混合起来：

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("square", type=int,
                    help="display a square of a given number")
parser.add_argument("-v", "--verbose", action="store_true",
                    help="increase output verbosity")
args = parser.parse_args()
answer = args.square**2
if args.verbose:
    print("the square of {} equals {}".format(args.square, answer))
else:
    print(answer)

可以看看运行结果：

$ python3 prog.py
usage: prog.py [-h] [-v] square
prog.py: error: the following arguments are required: square
$ python3 prog.py 4
16
$ python3 prog.py 4 --verbose
the square of 4 equals 16
$ python3 prog.py --verbose 4
the square of 4 equals 16
  • 注意顺序是无所谓的

那么如果我们进一步：

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("square", type=int,
                    help="display a square of a given number")
parser.add_argument("-v", "--verbosity", type=int,
                    help="increase output verbosity")
args = parser.parse_args()
answer = args.square**2
if args.verbosity == 2:
    print("the square of {} equals {}".format(args.square, answer))
elif args.verbosity == 1:
    print("{}^2 == {}".format(args.square, answer))
else:
    print(answer)

可以看到：

$ python3 prog.py 4
16
$ python3 prog.py 4 -v
usage: prog.py [-h] [-v VERBOSITY] square
prog.py: error: argument -v/--verbosity: expected one argument
$ python3 prog.py 4 -v 1
4^2 == 16
$ python3 prog.py 4 -v 2
the square of 4 equals 16
$ python3 prog.py 4 -v 3
16

我们这里面给他指定了一定的值后程序会怎么做。

这些看起来还不错，但是除了最后一个，不管输入什么，它都会返回相同的结果，我们想要严格限定他的输入必须是3：

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("square", type=int,
                    help="display a square of a given number")
parser.add_argument("-v", "--verbosity", type=int, choices=[0, 1, 2],
                    help="increase output verbosity")
args = parser.parse_args()
answer = args.square**2
if args.verbosity == 2:
    print("the square of {} equals {}".format(args.square, answer))
elif args.verbosity == 1:
    print("{}^2 == {}".format(args.square, answer))
else:
    print(answer)

可以看到此时就只能在choices=[0, 1, 2]中选择了：

$ python3 prog.py 4 -v 3
usage: prog.py [-h] [-v {0,1,2}] square
prog.py: error: argument -v/--verbosity: invalid choice: 3 (choose from 0, 1, 2)
$ python3 prog.py 4 -h
usage: prog.py [-h] [-v {0,1,2}] square

positional arguments:
  square                display a square of a given number

optional arguments:
  -h, --help            show this help message and exit
  -v {0,1,2}, --verbosity {0,1,2}
                        increase output verbosity
  • 注意这个只能在choices=[0, 1, 2]中选择这个信息不仅仅在报错error出显示了，还在我们的help text中显示了。

Now, let’s use a different approach of playing with verbosity, which is pretty common. It also matches the way the CPython executable handles its own verbosity argument (check the output of python --help):

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("square", type=int,
                    help="display the square of a given number")
parser.add_argument("-v", "--verbosity", action="count",
                    help="increase output verbosity")
args = parser.parse_args()
answer = args.square**2
if args.verbosity == 2:
    print("the square of {} equals {}".format(args.square, answer))
elif args.verbosity == 1:
    print("{}^2 == {}".format(args.square, answer))
else:
    print(answer)

We have introduced another action, “count”, to count the number of occurrences of a specific optional arguments:

$ python3 prog.py 4
16
$ python3 prog.py 4 -v
4^2 == 16
$ python3 prog.py 4 -vv
the square of 4 equals 16
$ python3 prog.py 4 --verbosity --verbosity
the square of 4 equals 16
$ python3 prog.py 4 -v 1
usage: prog.py [-h] [-v] square
prog.py: error: unrecognized arguments: 1
$ python3 prog.py 4 -h
usage: prog.py [-h] [-v] square

positional arguments:
  square           display a square of a given number

optional arguments:
  -h, --help       show this help message and exit
  -v, --verbosity  increase output verbosity
$ python3 prog.py 4 -vvv
16
  • Yes, it’s now more of a flag (similar to action="store_true") in the previous version of our script. That should explain the complaint.
  • It also behaves similar to “store_true” action.
  • Now here’s a demonstration of what the “count” action gives. You’ve probably seen this sort of usage before.
  • And if you don’t specify the -v flag, that flag is considered to have None value.
  • As should be expected, specifying the long form of the flag, we should get the same output.
  • Sadly, our help output isn’t very informative on the new ability our script has acquired, but that can always be fixed by improving the documentation for our script (e.g. via the help keyword argument).
  • That last output exposes a bug in our program.

Let’s fix:

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("square", type=int,
                    help="display a square of a given number")
parser.add_argument("-v", "--verbosity", action="count",
                    help="increase output verbosity")
args = parser.parse_args()
answer = args.square**2

# bugfix: replace == with >=
if args.verbosity >= 2:
    print("the square of {} equals {}".format(args.square, answer))
elif args.verbosity >= 1:
    print("{}^2 == {}".format(args.square, answer))
else:
    print(answer)

And this is what it gives:

$ python3 prog.py 4 -vvv
the square of 4 equals 16
$ python3 prog.py 4 -vvvv
the square of 4 equals 16
$ python3 prog.py 4
Traceback (most recent call last):
  File "prog.py", line 11, in 
    if args.verbosity >= 2:
TypeError: '>=' not supported between instances of 'NoneType' and 'int'
  • First output went well, and fixes the bug we had before. That is, we want any value >= 2 to be as verbose as possible.
  • Third output not so good.

Let’s fix that bug:

import argparse
parser = argparse.ArgumentParser()
parser.add_argument("square", type=int,
                    help="display a square of a given number")
parser.add_argument("-v", "--verbosity", action="count", default=0,
                    help="increase output verbosity")
args = parser.parse_args()
answer = args.square**2
if args.verbosity >= 2:
    print("the square of {} equals {}".format(args.square, answer))
elif args.verbosity >= 1:
    print("{}^2 == {}".format(args.square, answer))
else:
    print(answer)

We’ve just introduced yet another keyword, default. We’ve set it to 0 in order to make it comparable to the other int values. Remember that by default, if an optional argument isn’t specified, it gets the None value, and that cannot be compared to an int value (hence the TypeError exception).

And:

$ python3 prog.py 4
16

You can go quite far just with what we’ve learned so far, and we have only scratched the surface. The argparse module is very powerful, and we’ll explore a bit more of it before we end this tutorial.
